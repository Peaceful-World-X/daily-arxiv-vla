Card 02 论文概述

论文概述

  • 提出 Wan-Streamer v0.3，将原生流式交互模型重新框架化为一个统一的观点：视频是世界加上事件流
  • 世界是持久的环境、场景、角色、声音等稳定条件，事件流是世界内随时间变化的一切（行为、语音、环境变化等）
  • 基于此分解定义了一个通用预训练任务，并在实时全双工音视频交互任务上进行实例化，使模型能生成自由形式的语音和行为
Card 03 核心贡献

核心贡献

  • 将原生流式生成重新框架化为 世界+事件流 分解，分离持久世界上下文与时间变化事件，产生可跨实时任务迁移的通用预训练目标
  • 将模型实例化于实时全双工音视频交互，其中多模态理解过程类似 视觉-语言-行动（VLA），能将流式多模态输入映射为自然语言形式的语音和开放词汇行为动作
  • 展示此扩展保持了 v0.1/v0.2 的建模契约和 v0.2 的部署拓扑，保留了 640×368 分辨率、25 FPS、约 200 ms 模型侧延迟和约 550 ms 总交互延迟
Card 04 方法描述

方法描述

  • 采用 世界-事件分解 的数学形式化：世界是持久上下文的灵活结构化记录，事件是带时间区间、关联角色和自由形式描述的变化记录
  • 使用 流式世界-事件因式分解 进行自回归预测：给定世界和因果历史输入，实时预测下一个事件单元及其音视频实现
  • 创新点在于将持久世界上下文作为一级条件对象进行显式建模，并在交互中采用 角色扮演聊天格式，通过括号内的行为指令实现自由形式行为表达
Card 05 数据集与资源

数据集与资源

  • 使用 大规模、多样化视频语料库 进行预训练，每个视频剪辑提供推断的世界和带时间对齐的事件
  • 模型为 Wan-Streamer v0.3，性能参数保持与 v0.2 一致：640×368 视频分辨率、25 FPS 帧率、160 ms 流式单元
  • 训练和部署拓扑继承自 v0.2，使用 Thinker + Ulysses 上下文并行执行器 架构
Card 06 评估与结果

评估与结果

  • 评估重点在于 延迟和运行时协议：模型侧信号到信号延迟约 200 ms，总交互延迟约 550 ms（含 350 ms 双向网络预算）
  • 定性行为观察显示模型能在实时对话中同时发言并执行 开放词汇行为，行为指令在语言流中实现为连贯的面部表情、姿势变化、对声音的响应和与物体的交互
  • 行为保持与语音同步、与场景和角色身份一致，在无显式行为事件时维持稳定的空闲和聆听行为
